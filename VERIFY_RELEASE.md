# Priority 2 — Website

Before publishing **TDS v1.0.1**, review and verify the project website to ensure that all pages, links, and configuration settings are working correctly.

The following files and directories should be verified and updated, if necessary:

- `_config.yml` — Jekyll configuration
- `index.md` — Home page
- `about.md` — About page
- `CONTACT.md` — Contact page
- `404.html` — Custom error page
- `GITHUB_PAGES.md` — GitHub Pages documentation
- `_layouts/default.html` — Default page layout
- `assets/` — Images, styles, and other static resources

## Verification Checklist

Before proceeding to the next stage, confirm that:

- GitHub Pages is enabled and configured correctly.
- The website builds successfully.
- All internal links are working.
- Navigation menus are complete and functional.
- Static assets load correctly.
- The custom 404 page is available.
- Page metadata and permalinks are correct.
- No placeholder content remains.
